## 3. Pyrogram API ID / API Hash

- **الملف:** `bmqa/main.py` — السطر 62 تقريباً (استدعاء `Client(...)`)
- **القيمة:**
  - `api_id = 34366457`
  - `api_hash = "8a807fba11048941f9593da9fb9a59db"`
- **الخطورة:** عالية — هذه بيانات تطبيق تيليجرام (من my.telegram.org) وليست
  خاصة بحساب معين، لكنها مرتبطة بحساب المطوّر على my.telegram.org. تسريبها
  يسمح لأي شخص باستخدامها لإنشاء عملاء Telegram (MTProto clients) باسم هذا
  التطبيق، وقد يُستخدم لإرهاق الحصة (rate limits) أو أغراض إساءة تُنسب لحساب
  صاحب الـ API.
- **الإجراء المطلوب:**
  1. من https://my.telegram.org راجع تطبيقاتك.
  2. لا يوجد "تدوير" مباشر لـ api_id/api_hash من تيليجرام؛ إن كنت تشك بإساءة
     استخدام فعلية، الخيار الأضمن هو حذف التطبيق القديم وإنشاء تطبيق جديد
     للحصول على api_id/api_hash جديدين، ثم تحديث `.env`.

## 4. Session String لحساب Userbot (تم العثور على أثره فقط، تمت إزالة القيمة سابقاً)

- **الملف:** `bmqa/main.py` — بجانب السطر 62، يوجد تعليق بالعربي:
  > "تم حذف session string مسرب كان هنا (كان يمنح وصول كامل لحساب تيليجرام userbot)."
- **الخطورة:** حرجة جداً لو كانت القيمة القديمة لا تزال موجودة في أي نسخة/فرع/تاريخ
  Git سابق — أي شخص يملك session string يملك تحكماً كاملاً بحساب تيليجرام
  الشخصي (قراءة كل الرسائل، انتحال الهوية، الوصول لكل المحادثات الخاصة).
- **الإجراء المطلوب:**
  1. **مهم جداً:** إن كان هذا الـ session string قد ظهر فعلاً في أي نسخة سابقة
     من الكود (حتى لو حُذف لاحقاً)، فهو موجود في تاريخ Git وتعتبر الجلسة
     مخترقة تماماً بغض النظر عن الحذف الحالي.
  2. من إعدادات تيليجرام (Settings → Devices/Active Sessions) قم بإنهاء كل
     الجلسات النشطة لهذا الحساب فوراً ("Terminate all other sessions").
  3. أنشئ session string جديد يدوياً محلياً فقط، ولا تضعه إطلاقاً داخل أي ملف
     كود؛ ضعه في `.env` تحت `USERBOT_SESSION_STRING` فقط إن كنت فعلاً بحاجة
     userbot.
  4. راجع تاريخ Git بالكامل (`git log -p`) للتأكد من عدم وجود القيمة القديمة
     في أي commit سابق، واعتبر أي تاريخ فيه هذه القيمة يتطلب تنظيف/rewrite
     للـ history إن كان المستودع سيبقى عاماً.

## ملاحظة عامة

لم يتم العثور على كلمات مرور، مفاتيح قاعدة بيانات، أو مفاتيح سحابية (AWS/GCP/etc)
مكتوبة صراحة في بقية الملفات المفحوصة (`Plugins/*.py`, `helpers/*.py`,
`clean.py`, `Run`, `start.sh`, `waad-bot.service`, `README.md`). قيم مثل
`"password": ''` في `Plugins/downloader.py` (خاصة بخيارات `yt_dlp` لتسجيل
دخول OAuth2 لـ YouTube) فارغة وليست تسريباً.

بعد تدوير كل القيم أعلاه، تأكد من:
- عدم استخدام أي من القيم القديمة مرة أخرى.
- حذف أي نسخة قديمة من `config.py` / `information.py` تحتوي القيم الحقيقية
  من أي مكان (نسخ احتياطية، سيرفرات أخرى، محادثات، إلخ).
- إذا كان المستودع (bmqa الأصلي) قد رُفع لـ Git بأي وقت مع هذه القيم، فالتاريخ
  (git history) يحتفظ بها حتى لو حُذفت لاحقاً من آخر نسخة — يلزم إما جعل
  المستودع خاصاً بالكامل، أو تنظيف التاريخ (`git filter-repo` أو ما شابه)،
  بالإضافة إلى تدوير القيم نفسها (التدوير إلزامي بغض النظر عن حالة المستودع).
